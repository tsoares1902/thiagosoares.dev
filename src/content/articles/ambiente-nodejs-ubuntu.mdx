Um bom ambiente de desenvolvimento não precisa ser cheio de ferramentas.

Mas precisa ser previsível, confortável e fácil de manter.

Quando configuro uma máquina Linux para trabalhar com Node.js, gosto de pensar em alguns blocos:

- terminal;
- Git;
- Node.js;
- editor;
- Docker;
- ferramentas de API;
- acesso SSH;
- cloud.

## Atualizando o sistema

O primeiro passo é atualizar os pacotes.

```bash
sudo apt update
sudo apt upgrade -y
```

Também gosto de instalar alguns utilitários básicos.

```bash
sudo apt install -y curl git unzip build-essential ca-certificates gnupg
```

## Git

Git é obrigatório para praticamente qualquer projeto moderno.

```bash
git --version
```

Configuração inicial:

```bash
git config --global user.name "Thiago Soares"
git config --global user.email "tcsoares84@gmail.com"
```

## Node.js com NVM

Prefiro instalar Node.js usando NVM, porque isso permite alternar versões por projeto.

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
```

Depois:

```bash
nvm install --lts
nvm alias default lts/*
```

Verificação:

```bash
node -v
npm -v
```

## VS Code

Para trabalhar com Node.js, React, Next.js e NestJS, o VS Code continua sendo uma das melhores opções.

Extensões que costumo considerar úteis:

- ESLint;
- Prettier;
- Docker;
- GitLens;
- Tailwind CSS IntelliSense;
- Prisma;
- MongoDB for VS Code.

## Docker

Docker ajuda a manter serviços isolados.

Instalação no Ubuntu:

```bash
sudo apt install -y docker.io docker-compose-v2
```

Depois adiciono meu usuário ao grupo Docker.

```bash
sudo usermod -aG docker $USER
```

Depois disso, normalmente faço logout/login para aplicar a permissão.

Teste:

```bash
docker --version
docker compose version
```

## Exemplo de docker-compose

Um exemplo simples com MongoDB:

```yaml
services:
  mongodb:
    image: mongo:7
    container_name: mongodb-local
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db

volumes:
  mongodb_data:
```

Rodando:

```bash
docker compose up -d
```

## Ferramentas de API

Para testar APIs, gosto de ter pelo menos uma ferramenta dedicada.

Boas opções:

- Postman;
- Insomnia;
- Bruno;
- HTTPie.

Para terminal, HTTPie é bem agradável.

```bash
sudo apt install -y httpie
```

Exemplo:

```bash
http GET http://localhost:3000/health
```

## SSH

Para trabalhar com GitHub, servidores e cloud, configurar SSH é essencial.

```bash
ssh-keygen -t ed25519 -C "tcsoares84@gmail.com"
```

Depois posso copiar a chave pública.

```bash
cat ~/.ssh/id_ed25519.pub
```

## AWS CLI

Como uso AWS em projetos, também faz sentido instalar a CLI.

```bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
```

Verificação:

```bash
aws --version
```

## Organização de projetos

Gosto de manter uma pasta padrão para códigos.

```bash
mkdir -p ~/Code
```

Exemplo:

```bash
~/Code
├── fitematch-api
├── fitematch-site
├── fitematch-app
└── thiagosoares-dev
```

## Fechamento

Um ambiente Node.js profissional no Ubuntu não depende apenas de instalar Node.

Ele envolve uma combinação de terminal, Git, Docker, editor, chaves SSH e ferramentas de apoio.

Quando esse ambiente está bem configurado, fica muito mais simples trabalhar com APIs, aplicações Next.js, serviços NestJS e deploys em cloud.
