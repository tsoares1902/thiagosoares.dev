Node.js não surgiu apenas para permitir escrever JavaScript no backend.

Ele surgiu como resposta a um problema muito comum em aplicações web: lidar com muitas conexões simultâneas de forma mais eficiente.

## Antes do Node.js

Durante muito tempo, servidores web seguiram modelos onde cada requisição podia consumir uma thread ou processo.

Esse modelo funcionava, mas tinha um custo alto quando muitas conexões ficavam abertas ao mesmo tempo.

Imagine aplicações com:

- chats;
- uploads;
- streaming;
- APIs com muitas chamadas;
- integrações em tempo real;
- conexões long-lived.

O problema não era apenas executar código rápido.

O problema era esperar I/O.

## O nascimento do Node.js

Node.js foi criado por Ryan Dahl em 2009.

A ideia central era combinar:

- JavaScript;
- V8;
- I/O não bloqueante;
- Event Loop;
- APIs simples para rede e arquivos.

Essa combinação criou uma plataforma muito boa para aplicações orientadas a eventos.

## O papel da V8

A V8 é a engine JavaScript criada pelo Google para o Chrome.

Ela compila JavaScript para código de máquina e tornou possível executar JavaScript com ótima performance fora do navegador.

```txt
JavaScript
↓
V8
↓
Código de máquina
```

Node.js aproveitou essa engine para levar JavaScript ao servidor.

## I/O não bloqueante

O grande diferencial do Node.js é o modelo de I/O não bloqueante.

Em vez de travar a execução enquanto espera uma operação de arquivo, rede ou banco de dados, o Node.js continua trabalhando e executa o callback quando a operação termina.

Exemplo:

```ts
import fs from "node:fs/promises";

async function readConfig() {
  const content = await fs.readFile("config.json", "utf8");

  return JSON.parse(content);
}
```

Enquanto o arquivo é lido, o Node.js pode continuar lidando com outras tarefas.

## Event Loop

O Event Loop é o mecanismo que coordena a execução do código, callbacks, timers e operações assíncronas.

Ele é uma das razões pelas quais Node.js ficou tão popular para APIs e aplicações com muitas conexões.

<Callout>
Node.js não é mágico. Ele funciona muito bem quando o problema envolve bastante I/O, como APIs, bancos, arquivos, filas e integrações.
</Callout>

## npm e o ecossistema

Outro ponto decisivo foi o npm.

Com ele, a comunidade passou a compartilhar pacotes de forma muito simples.

Isso acelerou muito a adoção do Node.js.

```bash
npm install express
```

Com poucos comandos, era possível iniciar uma API, conectar um banco, validar dados e criar serviços completos.

## Node.js no mercado

Com o tempo, Node.js saiu do hype e virou uma tecnologia madura.

Hoje ele é usado para:

- APIs REST;
- BFFs;
- aplicações server-side;
- ferramentas de build;
- CLIs;
- microserviços;
- filas e workers;
- aplicações serverless.

## Por que eu gosto de Node.js

Gosto de Node.js porque ele permite construir soluções de ponta a ponta com uma stack moderna e produtiva.

Com TypeScript, NestJS, Docker, bancos como MongoDB ou PostgreSQL e cloud, é possível criar aplicações muito bem estruturadas.

## Fechamento

Node.js surgiu para resolver problemas reais de concorrência e I/O em aplicações web.

Mais do que “JavaScript no backend”, ele representa uma forma diferente de pensar servidores: orientada a eventos, não bloqueante e muito conectada ao desenvolvimento moderno de produtos digitais.
